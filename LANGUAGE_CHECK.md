# LANGUAGE_CHECK — 内部言語検証メモ

このファイルは制作・校閲用です。アプリ画面には表示しません。

## 1. 採用した言語層

中心は、チェンナイを含むタミル・ナードゥ州の都市部で地域横断的に通じやすい標準的な口語です。

採用した代表的な口語形：

- இருக்கு `irukku`：物の存在
- இருக்காங்க `irukkāṅka`：人の複数／敬意ある人の存在
- பாக்கறேன் `pākkaṟēṉ`：私は見る
- போறேன் `pōṟēṉ`：私は行く
- பண்றா／பண்றாங்க `paṇṟā / paṇṟāṅka`：する
- வேணும் `vēṇum`：必要だ／欲しい
- நாங்க `nāṅka`：私たち（通常、聞き手を含まない）
- நீங்க `nīṅka`：あなた／あなた方
- அவங்க `avaṅka`：あの人たち、または敬意ある「あの方」
- இதுக்கு／அதுக்கு `itukku / atukku`：これに／それ・あれに

極端に地域的な短縮、映画特有の乱暴な形、カースト・階層を強く示す形は採用していません。

## 2. 書記形との差

主な対応：

| 口語中心 | 丁寧・書記形の代表 | 備考 |
|---|---|---|
| இருக்கு | இருக்கிறது | 非人間単数・存在 |
| இருக்காங்க | இருக்கிறார்கள் | 人の複数／敬意 |
| பாக்கறேன் | பார்க்கிறேன் | 「見る」 |
| போறேன் | போகிறேன் | 「行く」 |
| பண்றாங்க | செய்கிறார்கள் | 「する」 |
| வேணும் | வேண்டும் | 必要・希望 |
| நாங்க | நாங்கள் | 排他的「私たち」 |
| நீங்க | நீங்கள் | 敬意／複数の二人称 |
| அவங்க | அவர்கள்／அவர் | 複数、または敬意単数 |
| இதுக்கு | இதற்கு | これに |
| அதுக்கு | அதற்கு | それ／あれに |

教材本文では口語形を学習対象とし、書記形は説明中の短い補足に限定しました。

## 3. ローマ字方式

基本は学習用の音韻的転写です。

- 長母音：`ā ī ū ē ō`
- そり舌音：`ṭ ṇ ḷ ḻ ṟ`
- 歯茎・歯音の細かな音価は、文字との対応を優先し `t / n` 系で表す
- `ṅ` は軟口蓋鼻音
- タミル文字 த は、環境により日本語話者には d 寄りに聞こえても、ローマ字では原則 `t`
- ローマ字は発音だけでなく構造表示を兼ねる

例：

- `vīṭu-kaḷ` 家-複数
- `vīṭṭ-il` 家の接続形-場所格
- `avaṉ-ai` 彼-対格
- `avaṉ-ukku` 彼-与格
- `pōṟ-ēṉ` 行く・口語現在-一人称単数

## 4. ハイフン分解の判断

### 名詞

- 基本形と複数：`vīṭu-kaḷ`
- 複数と格：`vīṭu-kaḷ-il`
- 接続形と格：`vīṭṭ-il`, `paṭatt-ai`
- 母音間のつなぎ：`kaṭai-y-il`, `cinimā-v-ukku`

### 代名詞

一・二人称の格形は単純な主格＋語尾ではないため、実際に用いられる接続形を基準に分けました。

- `eṉṉ-ai`：என்னை。対格形の表面に現れる重子音を保持
- `eṉ-akku`：எனக்கு。第一・第二人称単数に現れる与格形 `-akku`
- `uṉṉ-ai`, `uṉ-akku`
- `eṅkaḷ-ai`, `eṅkaḷ-ukku`

`நான்-ai` のような実在しない形は表示していません。

### 表面形と構造を分けた例

- புத்தகம் + -கள் → புத்தகங்கள்
  - 構造：`puttakam + -kaḷ`
  - 表面：`puttakaṅkaḷ`
- படம் → படத்து- + -இல் → படத்தில்
  - 構造：`paṭam → paṭatt(u)- + -il`
  - 表面：`paṭattil`

丸括弧の `(u)` は、接続形を説明するときの語末母音で、後続の母音始まり語尾の前では表面に出ないことを示します。

## 5. カタカナで再現しにくい音

- `ழ ḻ`：日本語のラ行ではない。舌を奥へ反らせた接近音
- `ள ḷ`：そり舌の lateral。便宜上「ル／ル̣」相当で表示
- `ற ṟ`：環境で速い r、強い t/r 系に聞こえる。口語現在形では弱くなる
- `ண ṇ`：そり舌鼻音。カタカナではナ行との差を十分に表せない
- `த t`：母音間で有声化し、ダ行寄りに聞こえることがある
- 語末の短い `u`：非常に弱くなる場合がある
- `ṅka`：日本語では「ンガ」に近いが、鼻音の調音位置は後続の k と同じ

カタカナは音読開始用です。音素の厳密な区別はタミル文字とローマ字に残します。

## 6. 文法上の判断

### 第1課

- 単数は無標としました。
- 複数の中心形を `-kaḷ` としました。
- மனித・敬意との関係がある `-r / -arkaḷ` は入口だけ示し、完全な人称・数・性体系は第14課へ送ります。
- `vīṭu → vīṭṭ(u)-`、`paṭam → paṭatt(u)-` を、斜格語幹の代表として採用しました。
- `kaṭai-y-il` の `y`、`cinimā-v-ukku` の `v` は、母音間の接続要素として示しました。
- 物の存在 `irukku` と人の複数 `irukkāṅka` を比較しました。
- `taṇṇīr` と `cāppāṭu` は、通常の文脈で数える個体ではなく量・集合として扱う入口を示しました。

### 第2課

- 主格を無標、対格を `-ai`、与格を `-ukku / -kku` としました。
- 人の直接目的語は原則として対格必須、非人間の不特定目的語は無標になり得る、という対比を入れました。
- 与格は受取人、移動の終点、経験者を一つの「到達点」概念で説明しました。
- `piṭikkum`, `vēṇum` は与格経験者構文として扱いました。
- 現代の一般的な指示体系では `itu` 対 `atu` を中心とし、日本語の「それ／あれ」の差は `atō` などの場所表現で補うと説明しました。
- 包括的 `nām` と排他的口語 `nāṅka` の対比を入れました。
- `avaṅka` は複数と敬意単数の両方を持つ口語形として扱いました。

## 7. 確認が必要な語形・表記

ネイティブ校閲で優先確認する項目：

1. 初級教材として `நண்பர்கள்` が口語中心の本文で硬すぎないか。より自然な日常語彙として `ஃப்ரெண்ட்ஸ்` を併記すべきか。
2. `போன் பண்றா／பண்றாங்க` の自然さと、地域横断的な中立性。`போன் பண்ணறா` 表記との比較。
3. `ஒரு நண்பர் எங்களுக்கு போன் பண்றார்` の口語・敬意の均衡。
4. `இந்தப் படத்தில் இரண்டு கார்கள் இருக்கு` を「映画に車が二台出る」と訳す文脈の自然さ。
5. カタカナの `படம்` を「パダム」、`புத்தகம்` を「プッタガム」とする有声化表示の一貫性。
6. `அவங்களை` のローマ字を `avaṅkaḷ-ai` と分析表示するか、口語表面に寄せ `avaṅkaḷai` と一続きにするか。
7. `இதுக்கு／அதுக்கு` を標準口語の中心に置く範囲と、丁寧な会話での `இதற்கு／அதற்கு` の頻度。
8. `நாம் ... போறோம்` が文脈なしで断定文と勧誘のどちらに読みやすいか。

未確認の語形を「唯一の正解」として扱わず、上記は承認済み見本化の前に母語話者または専門家の確認を受けます。

  - 斜格語幹、格、数の一般的整理の参照先。今回の直接例文は Schiffman の口語記述を優先。

## 9. 不確実性

- 「標準口語」は完全に固定された単一規範ではありません。都市部・教育・世代・場面によって、書記形寄りと縮約形寄りの幅があります。
- タミル文字で口語を綴る方法にも揺れがあります。TTSの挙動と教材上の見通しを優先し、過度な音声縮約は避けました。
- Web Speech APIのタミル語音声は端末・OS・音声エンジンに依存します。教材の語形が正しくても、合成音声の発音が地域的・書記語的になる可能性があります。
- カタカナは音韻情報を失います。承認後も、音声とローマ字を主、カタカナを補助とする方針を維持します。
