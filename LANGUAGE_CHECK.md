# 言語検証記録｜口語タミル語の教科書 v2.8

## 判定範囲

本版の公開教材範囲は独立参照ページ PART 0 と第1〜38章。PART 0 は名詞格、目的語標示、Tamil文字音を既習・未習を問わず引ける早見とする。第1〜4章は一人称、第5〜8章は丁寧二人称と疑問・応答、第9〜12章は第三者の性・敬称・複数と中性、第13〜16章はアスペクト・動詞連鎖・依頼／禁止・親称二人称／一人称複数、第17〜18章は引用・意図・伝聞・条件・譲歩・理由・連体節、第19〜28章は旅行10場面の語彙増殖と既習文法の再運用、第29〜38章は存在・所有・格・名詞・指示・数量・比較・焦点・名詞化・使役／態の補完。レジスターは教育を受けた話者間で通じるチェンナイ寄りの Standard Spoken Tamil。文語活用は読解補助または対照用の誤答を除き、正解データの主軸へ入れない。

## v1案からの修正

### 1. クラス差は過去だけではない

旧案の「現在 -r-・未来 -v- は共通、過去だけが分岐」は不採用。参照文法では、強変化は現在 `-kk(r)-`、未来 `-pp-`、不定詞 `-kka` を持つ。弱変化は現在 `-r- / -ur-`、未来 `-v-` 系。過去はさらに複数クラスへ分かれる。

採用例：

- `pō-r-ēn / pō-v-ēn / pōka`
- `paḍi-kk-r-ēn / paḍi-pp-ēn / paḍikka`
- `pāk-k-r-ēn / pā-pp-ēn / pākka`

### 2. 不定詞は機械的な -a ではない

採用形：`pōka, vara, sāppiḍa, vāṅka, pākka, paḍikka, irukka`。`pō` と `vā` は個別記憶、強変化は `-kka`。

### 3. 副動詞は機械的な -i / -u ではない

採用形：`pōyi, vandu, sāppiṭṭu, vāṅki, pāttu, paḍiccu, irundu`。過去1単と過去語幹を共有するペアとして提示する。

### 4. V-la は単純な過去否定ではない

参照文法の `pōka-lle` 型は時制を中和し、時間語により現在／過去を表す。教材では現代の口語綴り `போகல` 等を採り、機能ラベルを「非未来否定」とした。未来・意志否定は `V-a māṭṭēn` と分ける。

### 5. 単純現在と進行を同一視しない

`pōrēn` 等は現在、直近未来、習慣を文脈で表す。継続相・進行相は第13章で `AVP + -kkiṭṭu + iru` として別に扱う。

### 6. 自己表現は二レーン

- 行為・時制・意志：`nāṉ + finite verb + -ēn`
- 欲求・必要・能力：`eṉakku / eṉṉāle + infinitive + modal/stative`

`-ṇum` は「したい／必要／義務」を文脈でまたぐ。`eṉakku V-a āsai` を純粋な希望の補助表現として収録。`muḍiyum` は不定詞に続き、参照文法に従い `eṉakku` と `eṉṉāle` の意味差を注記した。

## v2.1 丁寧二人称・疑問文の追加検証

### 1. 丁寧二人称は -īṅka

Schiffman §3.5.5.1、§4.1 の全PNG表に従い、`nīṅka` に一致する有限動詞の語尾を `-īṅka(ḷ)` とした。単数の敬称と複数を兼ねる。初対面の旅行場面ではこの形を既定にし、親称 `nī + -ē` は使用域を学ぶまで正解例文の中心へ置かない。

| 辞書形 | 現在2敬 | 過去2敬 | 未来2敬 |
|---|---|---|---|
| `pō` | `pōrīṅka` | `pōnīṅka` | `pōvīṅka` |
| `vā` | `varrīṅka` | `vandīṅka` | `varuvīṅka` |
| `sāppiḍu` | `sāppiḍrīṅka` | `sāppiṭṭīṅka` | `sāppiḍuvīṅka` |
| `vāṅku` | `vāṅkurīṅka` | `vāṅkinīṅka` | `vāṅkuvīṅka` |
| `pāru` | `pākkrīṅka` | `pāttīṅka` | `pāppīṅka` |
| `paḍi` | `paḍikkrīṅka` | `paḍiccīṅka` | `paḍippīṅka` |
| `iru` | `irukkīṅka` | `irundīṅka` | `iruppīṅka` |

`iru` の現在には `irukk(r)īṅka` の変異がある。既存の一人称 `irukkēn` と統一し、本教材では r のない `irukkīṅka` を採用した。

### 2. 極性疑問は -ā、丁寧語尾の -ḷ が戻る

参照文法 §6.6.1 は、真偽を尋ねる疑問を通常「文の最後の要素＋`-ā`」で作り、主語・動詞の倒置を行わないとする。丁寧二人称語尾は単独では `-īṅka`、母音の疑問標識が続くと `-īṅkaḷ-ā?`。採用例は `pōrīṅkaḷā? / vandīṅkaḷā? / varuvīṅkaḷā?`。

Tamil文字では `போறீங்க → போறீங்களா?` と `ள` が現れる。教材の形態表示ではこれを `pō-r-īṅkaḷ-ā?` と分解し、音便を理由に未確認の短縮形へ置換しない。

### 3. 内容疑問は答え位置へ置く

疑問詞は `eṅge`「どこ」、`eppō`「いつ」、`eṉṉa`「何」、`enda + N`「どのN」、`yāru`「誰」、`eppaḍi`「どう」、`evvaḷavu`「いくら・どれほど」を採用。内容疑問では疑問詞を答えの入る位置へ置き、初級の無標の問いでは極性疑問 `-ā` を重ねない。

格を要する意味では `eṅge-irundu`「どこから」、`yār-e`「誰を」のように疑問詞側へ関係を保持する。旅行用例として `nīṅka eṅgirundu varrīṅka?`、`idu evvaḷavu?` を収録した。

### 4. 質問応答の変換

質問と回答の対応は、時制語幹を保存してPNGだけを切り替える。

- `pō-r-īṅkaḷ-ā? → pō-r-ēn`
- `pō-n-īṅkaḷ-ā? → pō-n-ēn`
- `pō-v-īṅkaḷ-ā? → pō-v-ēn`

否定回答は既存の二系統を維持し、過去・非未来には `V-la`、未来・意志には `V-a māṭṭēn` を用いる。

## v2.2 第三者・中性の追加検証

### 1. 親称の人物単数

Schiffman §4.1 の代名詞・一致表と、§3.5.5.1 の動詞パラダイムに従い、男性単数を `avan + -āṉ`、女性単数を口語 `ava + -ā` とした。完全代名詞 `avaḷ` は語彙表に残すが、会話例では口語 ava を主表示にする。

| 時制 | 男性 `pō` | 女性 `pō` | 男性 `paḍi` | 女性 `paḍi` |
|---|---|---|---|---|
| 現在 | `pōrāṉ` | `pōrā` | `paḍikkrāṉ` | `paḍikkrā` |
| 過去 | `pōnāṉ` | `pōnā` | `paḍiccāṉ` | `paḍiccā` |
| 未来 | `pōvāṉ` | `pōvā` | `paḍippāṉ` | `paḍippā` |

`avan / ava` は親称であり、未知の大人への無標の指示語として扱わない。教材例では身近な人物名 Ravi / Meena と結び付けた。

### 2. 敬称と複数の非対称性

男性敬称は `avaru + -āru`。女性敬称は `avaṅka + -āṅka` であり、後者は人物複数と同形になる。Schiffman のPNG表に合わせ、`avaru` を女性敬称へ機械転用しない。

| 用法 | 主語 | 現在 | 過去 | 未来 |
|---|---|---|---|---|
| 男性敬称 | `avaru` | `pōrāru` | `pōnāru` | `pōvāru` |
| 女性敬称 | `avaṅka` | `pōrāṅka` | `pōnāṅka` | `pōvāṅka` |
| 人物複数 | `avaṅka` | `pōrāṅka` | `pōnāṅka` | `pōvāṅka` |

女性敬称単数と人物複数は、先行する名詞、`reṇḍu pērum / ellārum / sila pēr` などの人数語、談話文脈で区別する。教材の診断問題でも、形だけから人数を一意に決めさせない。

### 3. 中性の三時制

Schiffman §4.1.2 は、中性PNGを一部不規則とし、未来 `-um` が人物未来標識ではなく不定詞側へ接続すると説明する。核7動詞は同文法の動詞パラダイムと照合し、次を採用した。

| 辞書形 | 中性現在 | 中性過去 | 中性未来 |
|---|---|---|---|
| `pō` | `pōradu` | `pōccu` | `pōkum` |
| `vā` | `varradu` | `vandadu` | `varum` |
| `sāppiḍu` | `sāppiḍradu` | `sāppiṭṭadu` | `sāppiḍum` |
| `vāṅku` | `vāṅkuradu` | `vāṅkinadu` | `vāṅkum` |
| `pāru` | `pākkradu` | `pāttadu` | `pākkum` |
| `paḍi` | `paḍikkradu` | `paḍiccadu` | `paḍikkum` |
| `iru` | `irukku` | `irundadu` | `irukkum` |

`pōccu` は完了した移動・出来事を報告する高頻度形、`irukku` は存在動詞の中性現在として独立登録した。`pōnadu` や `irukkudu` 等の変異は存在するが、本版の主表示へ混在させない。

### 4. 例文の意味制約

中性パラダイムは全核動詞を形一覧に収録する一方、他動詞へ不自然な無生物主語を強制しない。第12章の本文例は `vā / pō / iru` とバス・列車・店・本・水を中心にし、他動詞の中性形は「その物が〜する」のほか「〜すること」という名詞化的読みを持ちうる旨を活用表へ注記した。

## v2.3 動作拡張の追加検証

### 1. 進行・現在関連の完了・完遂

Schiffman のアスペクト記述と複合動詞節に従い、次の三つを別セルとして登録した。

| 機能 | 構造 | 採用例 | 診断上の核 |
|---|---|---|---|
| 進行 | `AVP + -kkiṭṭu + iru` | `sāppiṭṭukkiṭṭu irukkēn` | 出来事の途中 |
| 現在関連の完了 | `AVP + iru` | `vandirukkēn` | 結果・経験が現在に関係 |
| 完遂 | `AVP + (v)iḍu` | `sāppiṭṭuṭṭēn` | やり切り・区切り |

複合述語では、主動詞を非有限形へ置き、TAM・PNG は最後の `iru / (v)iḍu` 側へ一度だけ付ける。`(v)iḍu` の v は口語で脱落しやすいため、Tamil文字は `போயிட்டேன், வந்துட்டேன், சாப்பிட்டுட்டேன்` のような実用口語形を採用した。完遂形は後悔だけでなく、単純な完了・区切りにも使える旨を本文に明記した。

進行の綴りには分かち書き・縮約の変異がある。本版では構造が見える `சாப்பிட்டுக்கிட்டு இருக்கேன்` 系へ統一し、同一章内で `-கிட்டு / -க்கிட்டிரு` 等を無注記で混在させない。

### 2. 副動詞連鎖と目的不定詞

Schiffman の AVP 記述に基づき、順接・背景化を `V₁-AVP + V₂-finite`、目的を `V-infinitive + pō/vā` として対比した。

- `sāppiṭṭu pōrēn` — 食べてから行く
- `sāppiḍa pōrēn` — 食べるために行く
- `pōyi vāṅki varrēn` — 行って、買って、来る

一つの節として提示する例では、途中の動詞へ独立した有限語尾を重ねない。複数・第三者の例も最後の動詞だけを `-āṅka / -āṉ / -ā / -āru` にし、既習のPNG体系を保った。

### 3. 丁寧命令・否定命令・禁止

Penn の命令形教材と Schiffman §3.3 の imperative 記述に合わせ、旅行場面の既定命令を複数敬称 `stem + (u)ṅka` とした。採用形は `pōṅka, vāṅka, sāppiḍuṅka, vāṅkuṅka, pāruṅka, paḍiṅka, iruṅka`。特に次の語彙的最小対を分離した。

- `வாங்க vāṅka` — `vā`「来る」の丁寧命令「来てください」
- `வாங்குங்க vāṅkuṅka` — `vāṅku`「買う・受け取る」の丁寧命令

特定相手への丁寧な否定命令は `V-ā-d-īṅka`、一般的な不可・禁止は `V-a-k-kūḍādu`、自分の許可質問は `V-a-lām-ā?` とした。`pōkādīṅka` と `pōkakkūḍādu` を同一の日本語訳だけで扱わず、相手への働きかけ／一般規則の差を診断する。

### 4. 親称二人称と一人称複数

PNG表に従い、親称二人称単数は `nī + -ē`、一人称複数は `-ōm` とした。時制語幹は既習形を保持する。

| 主語 | 現在 `vā` | 過去 `vā` | 未来 `vā` |
|---|---|---|---|
| `nī` | `varrē` | `vandē` | `varuvē` |
| `nāma / nāṅka` | `varrōm` | `vandōm` | `varuvōm` |

包括 `nāma` は聞き手を含み、除外 `nāṅka` は聞き手を含まないが、動詞語尾はどちらも `-ōm`。参与範囲を学ぶ例では主語を省略しない。`nāma pōkalāmā?` は共同提案、`nāṅka pōkalāmā?` は聞き手を除く集団の許可質問として対比し、応答も `sari`／`āmā` とセットで示した。

## v2.4 引用・節拡張の追加検証

### 1. 引用標識 `-nnu` と埋込み

Schiffman §6.8、§7.3 に従い、独立して成立する文を `-nnu` の前へ置き、`sollu`「言う」、`kēḷu`「尋ねる／聞く」、`niṉai`「思う」などへ埋め込む構造を採用した。

- `nāṉ nāḷaikki pōvēn-nu soṉṉēn` — 明日行くと言った
- `avaru varuvārā-nnu kēṭṭēn` — あの方が来るか尋ねた
- `avaṅka eṅge pōrāṅka-nnu kēṭṭēn` — 彼らがどこへ行くか尋ねた

引用内の動詞は、外側の発話動詞が過去でもその時制へ機械的に合わせない。埋込み疑問も極性疑問 `-ā` または内容疑問詞を保持する。直接／間接の差では代名詞とPNGの視点が変わりうるため、本文では「引用内容をまず独立文として作る」手順を示した。

### 2. 意図 `-lām-nnu` と伝聞 `-ām`

Schiffman §6.8.2 の intent 構文に従い、`V-a-lām + -nnu + iru/niṉai` を採用した。`nāṉ pōkalām-nu irukkēn` は「行こうと思っている／行くつもりだ」。`-lām` 単独の勧誘・許可と区別し、最後の `iru/niṉai` がPNGを担う。

Schiffman §6.7 の reportive `-ām` は、話し手が真偽を直接保証せず「〜だそうだ／らしい」と報告する文末標識として分離した。`varuvārām` は伝聞、`varuvāru-nnu soṉṉāṅka` は引用内容＋発話動詞であり、同一セルにしない。

### 3. 条件 `-ā(l)` と譲歩 `-ālum`

Schiffman §6.10 に従い、条件形は過去語幹＋`-ā(l)` とした。口語の単独条件では終末の l が現れず、`vandā / pōnā / sāppiṭṭā / vāṅkiṉā / pāttā / paḍiccā / irundā` となる。条件形はTAMとPNGを中和するため、女性過去単数との表面上の同形性を本文で明示した。

譲歩は条件標識の `-āl` に `-um` を付ける。`vandālum`「来ても」、`eṅge pōnālum`「どこへ行っても」、`yāru vandālum`「誰が来ても」を採用した。否定条件は、口語で頻度の高い埋込み型 `vara-la-nnā` を拡張表へ置いた。

### 4. 理由形と連体形

Schiffman §6.8.4 の adjectival participle に従い、現在連体形を現在語幹＋`-a`、過去連体形を過去語幹＋`-a` とした。いずれもPNGを持たず、説明する名詞の前へ置く。

- `aṅge irukkara hōṭṭal` — そこにあるホテル
- `nāṉ pātta paḍam` — 私が見た映画
- `nāṉ vāṅkiṉa puttaham` — 私が買った本

理由は連体形＋中性名詞化 `-adu`＋原因・手段の `-nāle` と分析し、口語の `varradunāle / irukkaradunāle` を採用した。過去連体 `vanda` と条件 `vandā` は母音長と後続名詞の有無で対比する。

## v2.5 旅行科の追加検証

## v2.6 文法補完科の追加検証

### 1. 存在・所有・名詞文

`irukku / illa` は、所在、存在、状態、所有を前項の格と状態形から読む。所有は与格 `eṉakku` と人場所格 `eṉkiṭṭa` を分け、前者を一般的・抽象的な所有、後者を手元の所持の中心として提示した。境界は談話・方言で重なるため、排他的な規則にはしていない。

現在の単純な名詞文はゼロコピュラ、口語否定は `illa` を主表示にした。基盤書の機能表にある `alla` は誤りとはせず、書き言葉・改まった形の読解補助へ置いた。

### 2. 格・複数・名詞語幹

Schiffman の口語格記述と case stacking の分析に従い、次を採用した。

- 対格 `-e`：人の目的語、および特定・定の無生物。数詞付きの不特定無生物は無標識になりうる。
- 与格 `-kku`：受け手、行先、経験者、所有者。
- 場所 `-le / -kiṭṭa`：無生物場所と人物の所を分ける。
- 起点 `-lērundu / -kiṭṭērundu`：それぞれの場所形へ `-rundu` を積む。
- 共同 `-ōḍa / kūḍa`、手段 `-āle`。交通手段は日常口語の `basle / rayille` を主表示にする。

口語複数は有生名詞の `-ṅka(ḷ)` を中心にし、数詞後の無生物を機械的に複数化しない。人数は `numeral + pēr`。`vīḍu → vīṭṭukku / vīṭṭule` と代名詞 `nāṉ → eṉṉe / eṉakku / eṉṉōḍa` は、格接続時の斜格系列として登録した。

### 3. 指示・数量・比較

現代口語の指示体系は近称 `i-`、遠称 `a-`、疑問 `e-` の系列を主軸とした。日本語の「それ／あれ」をタミル語側で強制的に三分せず、旧い `u-` 系列は生成課題へ入れない。

数量は、個数 `ettaṉai`、量・価格・程度 `evvaḷavu`、人数 `pēr`、時刻 `maṇi-kku`、所要時間 `maṇi nēram` を分けた。序数は高頻度の `mudal` と生産的な `-āvadu` を採用した。

比較は形容詞を変化させず、基準を `B-e viḍa` で置く。最上級は比較範囲を場所格＋強調 `-le-yē` で明示する。範囲なしの機械的な「最上級語尾」は作らない。

### 4. 焦点・不特定

追加 `-um`、限定 `maṭṭum`、焦点 `tāṉ`、強い指定 `-ē` を同義扱いしない。疑問詞＋`-ō` は話し手に同定できない既存の人物・物・場所、疑問詞＋`-āvadu` は候補のどれか・自由選択を中心にした。疑問詞＋`-um`＋否定は `yārum illa` の全称否定として別セルにした。

### 5. 連体・名詞化・態

口語の正解生成は現在／非過去連体 `V-r-a`、過去連体 `past-a`、否定連体 `infinitive-āda` を中心にした。Schiffman の記述に従い、文語的な未来連体を口語パラダイムへ機械追加しない。名詞化は `-adu`、人物は `-avaṅka` を使い、出来事名詞化から `-adukku muṉṉāḍi / appuṟam` を作る。

使役は高頻度の自他動詞対と `V-a vai` を併用する。自己利益・自己管理は AVP＋`-kko`、相互は `oruttarukku oruttar` を採用した。受け身 `INF + paṭu` は存在するが会話の無標形とはせず、掲示・公的文書・報道の読解補助へ置く。日常会話では `āyiṭuccu / pōyiṭuccu / tiṟanduḍuccu` のような行為者なしの出来事報告を優先した。

### 6. v2.6 の人手確認対象

文法補完科は参照文法と既存の旅行例文を突き合わせ、全120例文・87文法型・40問を構造検査した。今後のチェンナイ話者確認では、次を優先する。

1. `eṉakku / eṉkiṭṭa` の所有差が各場面で自然か。
2. 借用語＋格の発音、特に `bassukku / basle` の表記と世代差。
3. 比較・焦点助詞が作る含意と、最上級の談話自然さ。
4. `-kko`、`V-a vai`、相互表現の縮約と語用。
5. 公的受け身を読解へ限定し、口語生成が過度に文語化していないか。

## v2.7 音声・転写精修

### 1. 書記形と口語音を別フィールドにする

従来は、慣用的なTamil書記形、書記形寄りのローマ字、口語の形態素分析、カナが一枚のカード内で無標識に並び、`டிக்கெட்டை / ṭikkeṭṭai / ṭikkeṭ-e / ティッケッテ` のように、学習者から見てどれが音なのか判別しにくい状態があった。v2.7では次の五層へ分離した。

1. `ta` — 慣用的なTamil書記形。
2. `spokenRoman` — ハイフンなしの口語音。
3. `morph` — 形態素境界を示す構造ローマ字。
4. `spokenKana` — 規格書01に照合したカナ。
5. `ttsText` — 端末TTSへ渡す音声用Tamil綴り。

表示順は `ta → spokenRoman → morph → spokenKana → 日本語` とし、構造行だけを設定で隠せる。口語ローマ字は常に残る。

### 2. 格語尾の採用音

Schiffman の Spoken Tamil 格表と Michigan State University *Basic Tamil* の口語対照に従い、次を主表示にした。

- 特定目的語：書記形 `-ஐ / -ai`、口語音 `-e`。
- 無生物の場所・交通手段：書記形 `-இல் / -il`、口語音 `-le`。
- 道具・原因：書記形 `-ஆல் / -āl`、口語音 `-āle`。
- 受け手・行先：`-(u)kku`。
- 起点：場所 `-le` に `-rundu` を重ねた `-lērundu`。
- 人の所：`-kiṭṭa`、同行：`-ōḍa / kūḍa`。

日本語の「で」や英語の `by / in / at` を一対一で格へ変換しない。場所・乗り物、道具・原因、同行という機能を判定してから形を選ぶ。

代表修正：

- `டிக்கெட்டை` — 書記形 `ṭikkeṭṭai`、構造 `ṭikkeṭṭ-e`、口語 `ṭikkeṭṭe`、カナ「ティッケッテ」。
- `அவரைப் பாத்தேன்` — 口語 `avare pāttēn`、カナ「アヴァレ パーッテーン」。
- `கடையில` — 口語 `kaḍele`、カナ「カデレ」。
- `கார்டால` — 口語 `kārḍāle`、カナ「カールダーレ」。
- `கத்தியால` — 口語 `kattiyāle`、カナ「カッティヤーレ」。

Tamil `ை` は規格書01の非語頭規則に従い、確認済みの借用語 `ஃப்ளைட் / வைஃபை / சைஸ் / டிரைவர்` と独立母音 `ஐயா` を保護した上で、口語ローマ字とカナを `e / エ段` へそろえた。`ரயில் rayil` の `ai` はTamil母音記号 `ை` ではなく文字列上の並びなので、機械置換から除外している。

### 3. 解説本文と音声

全38章の解説を走査し、Tamil文字を含む286箇所（268ユニーク）すべてに、既存データまたは手動確認済み索引から口語ローマ字とカナを付与した。直後にローマ字がすでに明記されている箇所は重複表示しない。

TTSは表示用 `ta` ではなく `ttsText || ta` を読む。これにより、書記形を教材から消さず、`-e / -le / -āle` を端末音声へ渡せる。ただし合成音声は母語話者録音の代替ではなく、端末ごとの発音差は残る。

### 4. 自動検査結果

転写対象1,285項目を検査し、次をゼロにした。

- 対格の構造 `-e` と口語ローマ字 `ai` の不一致。
- 手段格 `-āle` とローマ字／カナの `āla / アーラ` の不一致。
- 場所格 `-le` と旧カナ「〜ラ」「〜イレ」の不一致。
- 書記上の連声 `-p / -t / -k` が口語ローマ字へ残る例。
- 解説内でTamil文字だけになっている箇所。

`validate.mjs` は今後も五層の必須項目、代表回帰例、第30・31章の書記形→口語音表、本文注記の全件解決を公開前ゲートとする。

## v2.8 PART 0・文字音・カード全面再生

### 1. `-ai / -e / Ø` を同じ軸へ置かない

Tamil の対格について、PART 0 では次の二つの判断を分離した。

1. `-ai → -e` は、同じ対格の書記形から本教材の基準口語音への対応。
2. `Ø` は `-ai` の別発音ではなく、目的語を無標識にする統語・談話上の選択。

初学者向けの安全な出発点は、人・人名・代名詞、および指示詞などで特定された非人間目的語には対格を明示すること。非人間で不特定、種類、活動として捉える目的語では無標識を候補にする。これは日本語の「を」を機械的に省く規則ではなく、animacy と specificity が関与する differential object marking として扱う。実際の選択には語順、動詞、談話、方言・レジスターによる変異が残る。

対照例：

- `அந்தப் படத்தை பாத்தேன்` — `anda paḍatt-ai → anda paḍatte`。「その映画」という特定目的語なので対格を明示。
- `ஒரு படம் பாத்தேன்` — `oru paḍam pāttēn`。「ある映画一本」という非人間・不特定目的語で無標識が可能。
- `ரவியை பாத்தேன்` — `ravi-y-ai → raviye`。人名目的語なので対格を明示。

### 2. Tamil文字表

伝統的な12母音を横軸、18子音を縦軸とし、216の உயிர்மெய் எழுத்து を Unicode の子音字＋母音記号から生成する。独立母音、pulli を持つ純子音、各交点、ஃ、高頻度の借用文字を個別の `ta-IN` TTS入力にした。表はスマートフォンで横スクロールし、子音列を固定する。TTS出力は端末音声に依存するため、音素記述や母語話者録音の代替とはしない。

### 3. 例文カードの操作域

全ての再生可能な例文カードに同じ `data-play-id` を与え、Tamil本文、口語ローマ字、構造、カナ、日本語訳、余白のどこからでも既存の再生キューを起動する。丸い再生ボタンは視覚的な手掛かりとして残す。カード自体を `role="button"` とし、フォーカス時の Enter / Space にも対応する。

### 4. 互換性と検査

PART 0 は `book.chapters` の外に置き、URLは `#part=0` とした。既存の第1〜38章の番号、`#ch=N`、保存キー `tamil-verb-engine-v2`、章インデックス、完了記録を変更しない。自動検査では次を確認した。

- PART 0 の例文10文・確認4問・12×18＝216セル。
- カード本文タップと Enter による同一例文再生。
- 文字セル単独タップによる一文字TTS。
- PART 0 と既存38章を含む39セクションの印刷生成。
- 既存 `#ch=30` が引き続き第30章を開くこと。

## Tamil文字の表記方針

話し言葉には一つの強制的な正書法がない。本版は、読みやすさと形の可視性を優先し次を採用する。

- `போறேன், வர்றேன், சாப்பிடறேன், வாங்குறேன், பாக்கறேன், படிக்கறேன்`
- `போனேன், வந்தேன், சாப்பிட்டேன், வாங்கினேன், பாத்தேன், படிச்சேன்`
