# タミル語 文法の庭｜第1・2課 試作 v2

スマートフォンを中心に、タブレットでも読みやすい一列型の基礎文法アプリです。標準的な口語タミル語への入口を、例文の分解、音声、理由説明、映画会話への接続から学びます。

## ファイル構成

すべて同じ階層へ置きます。

- `index.html`
- `styles.css`
- `app.js`
- `lesson01.js`
- `lesson02.js`
- `README.md`
- `LANGUAGE_CHECK.md`
- `APPROVED_RULES.md`

第3課以降は、試作版の承認後に追加します。

## 開き方

1. 全ファイルを同じフォルダへ置きます。
2. `index.html` をSafari、Chrome、Edgeなどで開きます。
3. GitHub Pagesを使う場合も、追加のビルド処理は不要です。

Web Speech APIの実装と端末音声は、ブラウザ・OSによって異なります。タミル語音声が確認できない場合も、教材本文と問題は停止しません。

## 音声操作

- タミル文字の行をタップ：そのタミル語だけを再生
- 下部の設定：速度と繰返し回数を共通適用
- `課内を再生`：現在の課にある、展開中のタミル語項目を順番に再生
- `■`：停止

TTSへ渡すのは `tts` に保存した自然なタミル文字列だけです。日本語、ローマ字、カタカナ、文法解説、ボタン名は読み上げません。

## v2で変更したこと

- 660〜1099pxのタブレット幅では、長い例文カードを無理に二列化しない
- 広いデスクトップだけ、内容に余裕がある箇所を二列化する
- トロピカル装飾を、背景グラデーション中心から、海・日差し・葉・コーラムの線的表現へ整理する
- 文法解説を「規則」だけでなく、「なぜその形が必要か」「映画で何が拾えるか」まで拡張する
- 映画・チェンナイ文化の補足を、文法本文と区別した濃紺の読み物カードにする
- 表面形と構造形の差が大きい語に、`surface` 表示を追加する
- タミル語音声を確認できない場合の短い状態表示を追加する

## 教材データ

各課は `window.TAMIL_LESSONS` へデータを追加します。主要な例文項目は次のフィールドを使います。

- `tamil`：画面表示するタミル文字
- `tts`：TTSへ渡すタミル文字
- `roman`：文法構造を示すローマ字
- `surface`：必要な場合だけ表示する表面発音形
- `katakana`：日本語話者向けの近似
- `japanese`：自然な日本語訳
- `literal`：必要な場合だけ表示する構造寄りの訳・注記
- `morph`：形態素の短い説明
- `context`：自宅、映画、旅行などの場面タグ

## 検証の優先順位

1. タミル語母語話者による例文の自然さ
2. 口語形と教材用の保守的綴りの距離
3. ローマ字分解と実際の表面音の整合
4. iPhone SafariでのTTS
5. iPad縦・横画面でのカード密度
6. 第1課・第2課の長さと、途中から再開しやすいか
