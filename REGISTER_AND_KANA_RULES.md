# 標準口語・カナ表示規則 v2

このファイルは、全20課の主表示、補助表示、カタカナ転写を横断的に統一する基準である。既存の暫定規則と衝突する場合は、本ファイルを優先する。

## 1. 三層のレジスター

### 主表示：標準口語

タミル・ナードゥ州の都市部を中心に広く理解され、特定地域の俗語に寄りすぎない会話形を主教材とする。

代表例：

- `பார்க்கிறேன் → பாக்கறேன்`
- `போகிறேன் → போறேன்`
- `வருகிறேன் → வர்றேன்`
- `இருக்கிறது → இருக்கு`
- `வேண்டும் → வேணும்`
- `நாங்கள் / நீங்கள் / அவர்கள் → நாங்க / நீங்க / அவங்க`
- `இன்று / நேற்று / நாளை → இன்னிக்கு / நேத்து / நாளைக்கு`
- `என்று → -ன்னு`

極端に地域的な俗語、乱暴な映画表現、カースト・階層を強く示す形は主表示へ入れない。

### 補助1：チェンナイの耳

チェンナイの会話・映画で特に聞こえやすく、主表示との差が聞き取りに役立つ場合だけ掲載する。

例：

- `பாக்கறேன் → பாக்குறேன்`
- `வர்றேன் → வரேன்`
- `செய்றேன் → பண்றேன்`
- `இல்லனா → இல்லேனா`
- `வந்த அப்புறம் → வந்தப்புறம்`

### 補助2：書記形

字幕、看板、文章、ニュース、書籍を読むために重要な保守形・書記形を小さく掲載する。書記形を主表示へ無表示で混ぜない。

例：

- `பாக்கறேன் ↔ பார்க்கிறேன்`
- `போறேன் ↔ போகிறேன்`
- `இருக்கு ↔ இருக்கிறது`
- `வேணும் ↔ வேண்டும்`
- `-ன்னு ↔ என்று`

## 2. 一つの例文カードの順序

1. 標準口語のタミル文字
2. 規格準拠カタカナ
3. 文法構造が見えるローマ字
4. 自然な日本語
5. 必要な場合だけチェンナイ形・書記形

TTSへ渡すのは主表示の自然なタミル文字だけである。

## 3. カタカナの役割

カタカナは、音読開始と音の想起を助ける補助表記である。タミル語の音韻情報を完全に保存するものではない。失われる区別はタミル文字、構造ローマ字、音声が担う。

同じタミル文字列からは、常に同じカタカナを生成する。ローマ字を見てカタカナを作らず、タミル文字と音環境から判定する。

## 4. カタカナの主要裁定

- 長母音は長音符 `ー` で示す。
- `வ` はヴァ行で統一する。
- `ழ` はラ行で近似し、ザ行にしない。
- 語頭の独立母音 `எ / ஏ` は `イェ / イェー`。
- 非語頭の母音記号 `ை` は原則としてエ段で近似する。
- 重子音は `ッ` で示す。
- 鼻音＋破裂音は `ン＋濁音` を基本とする。
- 母音間の `க / ட / த / ப` は、実際の有声化をカナへ反映する。
- `ற்ற` は `ットゥラ／ットゥリ` 系として処理する。
- 語末の短い母音や子音は、カナで過剰に精密化しない。

固定例：

- `வணக்கம்` → ヴァナッカム
- `என்ன` → イェンナ
- `எனக்கு` → イェナック
- `சென்னை` → チェンナイ
- `வேணும்` → ヴェーヌム
- `சொல்லுங்க` → ソッルンガ
- `வாழைப்பழம்` → ヴァーレッパラム
- `புத்தகம்` → プッタガム
- `படம்` → パダム
- `போறேன்` → ポーレーン
- `பாக்கறேன்` → パーッカレーン

## 5. 文法説明の基準

各課は、用語の定義より先に次を示す。

1. 何を区別するための形か
2. 文のどこに現れるか
3. 形をどこで切るか
4. 似た形と何が違うか
5. 日本語と同じ点・異なる点
6. 旅行・映画で何を聞き取れるか

長い文は、最後の有限述語を先に見つけ、格語尾、非定形、引用、条件・理由・時間の節末標識から逆算する。

## 6. 実装

- `lesson01.js`〜`lesson20.js`：原教材データ
- `lesson-revision.js`：全課へ標準口語・カナ・課の芯を横断適用
- `lesson-ui-revision.js`：カード順序と補助レジスターを表示
- `revision.css`：補助表示の共通スタイル

この構成により、全課へ同じ裁定を適用し、個別ファイルでの転写揺れを防ぐ。

## 7. 残る確認

- 標準口語とチェンナイ形の境界は、母語話者確認を経て調整する。
- カタカナは規則適用後も、全主要例文を音声と照合する。
- 書記形から標準口語への変換が意味や敬意を変えていないか確認する。
- TTSが口語綴りを不自然に読む場合は、表示用 `tamil` と `tts` を個別に分ける。
